# Screenshot capture guide — TrainerApp

Play Console needs **at least 2 phone screenshots** (use 4–6).  
Target size: portrait **1080 × 1920** (or any 9:16 between 320px and 3840px). Format: PNG or JPEG.

Save final files into:

```
store-assets/screenshots/
```

## Capture method (recommended)

1. Open the installed Android app (or Chrome device mode at 1080×1920).
2. Sign in with a clean demo trainer account that has 2–3 sample clients and one logged session.
3. Take screenshots on device (`Power + Volume Down`) or Android Studio emulator.
4. Do **not** include real client phone numbers or private medical notes in marketing shots.
5. Optional: add a thin mint caption bar later in Figma — keep UI readable.

## Do / Don't

- Do use India-friendly sample names and INR-style payment notes if shown.
- Do keep status bars clean; hide debug banners.
- Don't show admin console, raw API errors, or Suspended billing screens.
- Don't upload tablet screenshots unless you also declare tablet support.

## After capture

Upload under **Play Console → Grow → Store presence → Main store listing → Phone screenshots**.
